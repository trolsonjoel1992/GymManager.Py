==========================================
GESTOR DE GIMNASIO - PROYECTO ACADÉMICO
==========================================

ENUNCIADO DEL PROYECTO
-----------------------
Desarrollar un sistema en Python para la gestión de un gimnasio que permita:

- Administrar socios (registro, control de cuotas, inscripción a actividades).
- Controlar asistencia a actividades.
- Generar estadísticas básicas de concurrencia.
- Gestionar distintos tipos de membresías y promociones.
- Persistencia de datos mediante archivos JSON (u otro formato simple).
- Interfaz por consola (menú de opciones).
- Código minimalista, modular y fácil de comprender.

====================================================================

LO REALIZADO HASTA EL MOMENTO (FASE 1: GESTIÓN DE SOCIOS)
----------------------------------------------------------

Se ha completado el módulo de **ABM de socios** (altas, bajas, modificaciones y listado) con las siguientes características:

1. **Estructura de carpetas modular**:
   - `domain/`     → entidades del negocio (clase Socio).
   - `services/`   → lógica de negocio (gestor de socios).
   - `persistence/`→ manejo de archivos JSON.
   - `utils/`      → validadores genéricos.
   - `ui/`         → interfaz de usuario por consola (inputs, menús).
   - `files/`      → almacenamiento de datos (socios.json).

2. **Clase Socio (domain/socio.py)**:
   - Atributos: id (autogenerado), dni (clave única), nombre_completo, teléfono, dirección, email, fecha_inscripción (automática), activo (borrado lógico).
   - Métodos `to_dict()` y `from_dict()` para serialización JSON.

3. **Persistencia (persistence/repositorio.py)**:
   - Funciones `cargar_socios()`, `guardar_socios()`, `existe_dni()`.
   - Archivo: `files/socios.json`.

4. **Servicio de socios (services/gestor_socios.py)**:
   - `alta_socio()` → crea nuevo socio con ID incremental, verifica DNI único.
   - `listar_socios()` → retorna solo activos (o todos opcionalmente).
   - `buscar_por_dni()`, `buscar_por_id()`.
   - `editar_socio()` → permite modificar nombre, teléfono, dirección, email (no DNI ni id).
   - `eliminar_socio_logico()` → cambia activo a False.
   - Todas las funciones devuelven tupla `(éxito, dato, mensaje)` para manejo en UI.

5. **Validaciones (utils/validadores.py)**:
   - DNI: 7-8 dígitos numéricos.
   - Email: formato usuario@dominio.ext.
   - Teléfono: 9-15 dígitos, opcional '+' inicial.
   - Nombre: no vacío y sin números.

6. **Interfaz de usuario (ui/)**:
   - `ui/inputs.py`: función genérica `solicitar_dato()` con validador opcional y reingreso automático.
   - `ui/menu_socios.py`: funciones para agregar, listar, editar y eliminar socios, integrando solicitud de datos y mensajes.
   - `main.py`: menú principal con bucle de opciones.

7. **Flujo completo de gestión de socios**:
   - Registro de socio con validación campo a campo.
   - Edición con campos opcionales (dejar vacío para no modificar).
   - Baja lógica (no se elimina físicamente).
   - Listado de socios activos con todos sus datos visibles.

8. **Persistencia funcional**:
   - Los datos se guardan y recuperan correctamente en `files/socios.json`.
   - El sistema crea automáticamente la carpeta `files/` si no existe.
