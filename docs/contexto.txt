1. DESCRIPCIÓN GENERAL
-------------------------------------------------------------------------------
Aplicación de gestión para un gimnasio, desarrollada en Python, que corre en 
consola. Permite administrar socios, actividades, inscripciones, asistencias, 
pagos y generar estadísticas. Sigue una arquitectura por capas (dominio, 
persistencia, servicios, UI, utilidades) y utiliza archivos JSON como base de 
datos.

3. MODELOS DE DATOS
-------------------------------------------------------------------------------
Socio (domain/socios.py):
- numero_socio: int (autoincremental)
- dni: str (único, 7-8 dígitos)
- nombre_completo: str
- telefono: str
- direccion: str
- email: str
- membresia: Literal["basica", "premium"]
- fecha_inscripcion: date (hoy por defecto)
- fecha_ultimo_pago: Optional[date] (None si no hay pagos)
- activo: bool (True por defecto)
- motivo_baja: Optional[Literal["mora", "manual"]] (None si activo)

Pago (domain/pagos.py):
- id: int (autoincremental)
- numero_socio: int (referencia a Socio)
- fecha_pago: date
- monto: float
- meses_cubiertos: int (1,2,3,...)
- observaciones: str (opcional)

4. LÓGICA DE NEGOCIO (SOCIOS)
-------------------------------------------------------------------------------
- Vencimiento de cuota: fecha_ultimo_pago + 1 mes (o fecha_inscripcion + 1 mes si no hay pago).
- Estados del socio (según función obtener_estado()):
  - "activo": hoy <= vencimiento. Socio al día.
  - "debe_cuota": vencimiento < hoy <= vencimiento + 10 días. Período de gracia.
  - "inactivo_por_deuda": hoy > vencimiento + 10 días. El sistema desactiva automáticamente.
  - "inactivo": baja manual (motivo_baja = "manual").
- Baja automática: se ejecuta al inicio de main.py. Desactiva socios con más de 10 días desde el vencimiento.
- Reactivación:
  - Si motivo_baja == "manual": paga 1 mes.
  - Si motivo_baja == "mora": calcula fecha de baja = vencimiento + 10 días.
    - Si misma mes y año que hoy → paga 1 mes.
    - Si no → paga 2 meses (mes vencido + mes actual, pero solo se extiende 1 mes).
  - Al reactivar: activo=True, motivo_baja=None, fecha_ultimo_pago = hoy + 1 mes.
  - Se registra un pago automático con meses_cubiertos = 1 o 2 (según el caso).

5. LÓGICA DE NEGOCIO (PAGOS)
-------------------------------------------------------------------------------
- Pago manual (socio activo):
  - Se extiende la fecha_ultimo_pago sumando los meses pagados.
  - Si la nueva fecha < hoy, se ajusta a hoy + meses.
  - Se registra el pago en el historial.
- Pago automático (para altas y reactivaciones):
  - Se crea un pago con monto = 0.0 y meses_cubiertos = según corresponda.
  - Se usa para mantener trazabilidad.
- Un socio inactivo NO puede registrar pagos manuales; debe reactivarse primero.

6. FLUJOS DE USUARIO PRINCIPALES (MODULOS SOCIOS Y PAGOS)
-------------------------------------------------------------------------------
MÓDULO SOCIOS (menu_socios.py):
- Registrar socio:
  1. Ingresar DNI.
  2. Si existe activo → mensaje y vuelve.
  3. Si existe inactivo → ofrece reactivar.
  4. Si no existe → pedir resto de datos, crear socio, registrar pago de alta.
- Listar socios (submenú):
  1. Todos (activos e inactivos)
  2. Morosos (período de gracia)
  3. Activos
  4. Inactivos (con motivo)
  5. Ver detalle (búsqueda por DNI o número)
- Editar socio: buscar por identificador (DNI o número), modificar campos.
- Baja lógica: buscar por identificador, marcar activo=False, motivo_baja="manual".
- Reactivar: buscar por identificador, mostrar info, confirmar, ejecutar.

MÓDULO PAGOS (menu_pagos.py):
- Registrar pago:
  1. Buscar por identificador.
  2. Si inactivo → ofrecer reactivar (si confirma, ya registra pago y finaliza).
  3. Si activo → pedir monto y meses, actualizar cobertura, guardar pago.
- Ver estado de cuotas: buscar por identificador, mostrar estado y vencimiento.
- Ver historial de pagos: buscar por identificador, listar pagos.

11. CONSIDERACIONES IMPORTANTES
-------------------------------------------------------------------------------
- El identificador "DNI o número de socio" es un string que puede ser numérico (número de socio) o alfanumérico (DNI).
- La función buscar_por_identificador() primero intenta como número (si es dígitos) y luego como DNI.
- Los pagos de altas y reactivaciones tienen monto = 0.0 para distinguirlos de pagos manuales.
- El estado "moroso" se refiere al período de gracia (10 días posteriores al vencimiento).
- El sistema desactiva automáticamente a los socios que superan los 10 días de gracia, al iniciar el programa.
- La reactivación desde el registro de pago evita duplicar el pago: ya se registra el pago de reactivación, por lo que no se pide monto y meses adicionales.
